Section 01
Anatomy of an Ideal Prompt

Every great prompt is built from five foundational components. Miss one and you leave Claude guessing.

ROLEYou are a senior Python developer specializing in data pipelines.
CONTEXTI have a CSV file with 500k rows. It has columns: user_id, event_type, timestamp, value. The pipeline currently takes 45 minutes.
TASKRewrite the processing function to use pandas vectorized operations instead of row-by-row iteration. Focus on the aggregate step.
FORMATProvide: (1) the refactored function, (2) a before/after comparison, (3) expected speedup estimate.
CONSTRAINTSUse only pandas and numpy. Python 3.10+. No external dependencies. Preserve the existing function signature.

Role

Sets Claude's perspective, expertise level, and behavioral frame for the entire conversation.

Context

Provides the background, current state, data shape, or situation Claude needs to understand your problem.

Task

The explicit, specific action you want Claude to perform. Use strong verbs: rewrite, analyze, compare, generate.

Format

Desired output structure: numbered list, code block, table, JSON, essay, bullet points, step-by-step.

Constraints

Boundaries, limitations, must-haves, and must-avoids that prevent incorrect assumptions.

Section 02
How Claude Processes Your Prompt

Behind every response is a multi-stage internal process. Understanding it helps you write prompts that align with how the model thinks.

01

Tokenization

Your prompt is split into tokens (roughly word-pieces). Context window size limits total tokens in + out.

~milliseconds
02

Intent Parsing

Claude identifies the core request type: question, generation, transformation, analysis, or conversation.

semantic layer
03

Context Weighting

Attention mechanisms assign weight to different parts of your prompt. Earlier context and clear specifics get higher weight.

attention
04

Knowledge Retrieval

Relevant training knowledge is activated and weighted against your specific context and constraints.

parametric memory
05

Chain-of-Thought

For complex tasks, internal reasoning steps are generated (especially with "think step by step" prompting).

reasoning
06

Format Assembly

The response is structured according to detected format signals: code blocks, headers, lists, or prose.

output structure
07

Token Generation

Output tokens are generated autoregressively — each word influences the next. Precision in your prompt = precision in output.

autoregressive
💡
Why this matters for prompting

Claude doesn't "understand" your prompt the way a human would — it computes probability distributions over tokens. This means ambiguous prompts produce unpredictable outputs. Every specific detail you provide narrows the distribution toward your intended answer. Structure signals (like "respond in JSON" or "step by step") directly influence the format assembly stage.

Section 06
10 Best Practices

Proven techniques that dramatically improve output quality.

🎭

Assign a Role

Start with You are a [expert]. It frames Claude's knowledge activation and writing style from the first token.

🧩

One Task Per Prompt

Avoid "write X, then analyze Y, then suggest Z." Break compound requests into separate prompts for each goal.

📐

Use Positive + Negative Constraints

Tell Claude what TO do AND what NOT to do. Do NOT use bullet points is as powerful as specifying format.

🔢

Request Step-by-Step Reasoning

Add Think step by step or Reason before answering for complex tasks. This activates chain-of-thought internally.

📎

Provide a Sample Output

One concrete example of the style, format, or quality you want is worth 10 adjectives describing it. Show, don't just tell.

🏷️

Use XML Tags for Structure

Wrap distinct sections in tags: <context>, <task>, <format>. Claude parses these cleanly, especially in long prompts.

📏

Specify Exact Length

"Write a summary" is vague. "Write a 3-sentence summary under 80 words" is a directive. Length specificity directly controls verbosity.

🎯

Name Your Audience

"Explain to a 5-year-old" and "Explain to a PhD student" produce radically different outputs from identical tasks. Audience is a force multiplier.

🔄

Iterate, Don't Restart

Use follow-up prompts like Revise section 2 to be more concise rather than rewriting the whole prompt. Claude holds conversation context.

⚠️

State the Failure Mode

Tell Claude what a bad response looks like: If you're uncertain, say so rather than guessing. This sets quality expectations explicitly.

Section 07
Quick Reference Cheat Sheet

Power phrases and patterns that reliably improve prompt performance.

Tone & Style
Write in a direct, conversational tone — no corporate jargon
Use active voice throughout
Avoid adverbs like "very", "extremely", "really"
Sound like [specific person or brand] — not like AI
Match the energy of the audience: [describe them]
Output Format
Respond only in JSON with keys: [list them]
Use markdown with H2 headers and bullet points
Output as a numbered list with max 1 sentence per item
Start with a 1-line summary, then details below
Return only the code — no explanation unless asked
Reasoning Triggers
Think step by step before answering
First identify what information is missing
Before responding, restate the problem in your own words
Consider edge cases and mention any assumptions
If there are multiple valid approaches, list them first
Guardrails & Quality
Do NOT start with "Certainly!", "Great question!", or filler phrases
If unsure, say "I'm not certain" rather than guessing
Do NOT pad the response with obvious context I already know
After responding, rate your own confidence (1–5) with a reason
Cite any statistics you include — do not invent numbers
