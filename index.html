Section 01

Anatomy of an Ideal Prompt

Every great prompt is built from five foundational components. Miss one and you leave Claude guessing — and guessing costs you revision cycles.

ROLE You are a senior Python developer specializing in data pipelines.
CONTEXT I have a CSV file with 500k rows. Columns: user_id, event_type, timestamp, value. The pipeline currently takes 45 minutes to run.
TASK Rewrite the processing function to use pandas vectorized operations instead of row-by-row iteration. Focus on the aggregate step.
FORMAT Provide: (1) the refactored function, (2) a before/after comparison, (3) expected speedup estimate.
CONSTRAINTS Use only pandas and numpy. Python 3.10+. No external dependencies. Preserve the existing function signature.

Role

Sets Claude's persona, expertise level, and behavioral frame for the entire response.

Context

The background, current state, data shape, or situation Claude needs to understand your problem.

Task

The explicit, specific action you want. Use strong verbs: rewrite, analyze, compare, generate.

Format

Desired output structure — numbered list, code block, table, JSON, essay, bullet points.

Constraints

Boundaries, limitations, must-haves and must-avoids that stop incorrect assumptions.

Section 02

How Claude Processes Your Prompt

Behind every response is a multi-stage internal process. Understanding it helps you write prompts that align with how the model actually thinks.

Claude's Internal Processing Pipeline
flowchart LR
  A([📝 Your Prompt]) --> B[🔢 Tokenization]
  B --> C[🎯 Intent Parsing]
  C --> D[⚖️ Context Weighting]
  D --> E[📚 Knowledge Retrieval]
  E --> F[🧠 Chain-of-Thought]
  F --> G[🔧 Format Assembly]
  G --> H([💬 Response])

  style H fill:#1d9e75,color:#fff,stroke:#1d9e75

Tokenization

Your prompt is split into tokens (roughly word-pieces). Context window limits total tokens in + out.

~milliseconds

Intent Parsing

semantic layer

Attention mechanisms assign weight to different parts of your prompt. Specific details get higher weight.

04

Knowledge Retrieval

Relevant training knowledge is activated and weighted against your specific context and constraints.

parametric memory
05

Chain-of-Thought

For complex tasks, internal reasoning steps are generated — especially when you say "think step by step".

reasoning
06

Format Assembly

The response is structured according to detected format signals: code blocks, headers, lists, or prose.

output structure
07

Token Generation

Output tokens are generated autoregressively — each word influences the next. Precision in = precision out.

autoregressive
💡
Why this matters for prompting

Claude doesn't "understand" your prompt the way a human would — it computes probability distributions over tokens. This means ambiguous prompts produce unpredictable outputs. Every specific detail you provide narrows the distribution toward your intended answer. Structure signals (like "respond in JSON" or "step by step") directly influence the Format Assembly stage and lock in the shape of the response.

Section 06

10 Best Practices

Proven techniques that reliably improve output quality — pick any one and your prompts get better immediately.

🎭

Assign a Role

Start with You are a [expert]. It frames Claude's knowledge activation and writing style from the first token generated.

🧩

One Task Per Prompt

Avoid "write X, then analyze Y, then suggest Z." Break compound requests into separate prompts. Multi-goal prompts dilute focus.

📐

Use Positive + Negative Constraints

Tell Claude what TO do AND what NOT to do. Do NOT use bullet points is as powerful as specifying the format directly.

🔢

Request Step-by-Step Reasoning

Add Think step by step for complex tasks. This activates chain-of-thought internally and dramatically improves accuracy.

📎

Provide a Sample Output

One concrete example of the style or quality you want is worth ten adjectives describing it. Show, don't just tell.

🏷️

Use XML Tags for Structure

Wrap sections in tags: <context>, <task>, <format>. Claude parses these cleanly in long prompts.

📏

Specify Exact Length

"Write a summary" is vague. "Write a 3-sentence summary under 80 words" is a directive. Length specificity controls verbosity.

🎯

Name Your Audience

"Explain to a 5-year-old" vs "Explain to a PhD" produces radically different outputs from the same task. Audience is a force multiplier.

🔄

Iterate, Don't Restart

Use follow-ups like Revise section 2 to be more concise rather than rewriting the whole prompt. Claude holds conversation context.

⚠️

State the Failure Mode

Tell Claude what a bad response looks like: If you're uncertain, say so rather than guessing. Sets quality expectations explicitly.

Section 09 — Troubleshoot

Common Mistakes & Fixes

Even experienced users make these mistakes. Each one is easy to fix once you know what to look for.

01 Asking Claude to fix code without sharing the code
▾

Claude cannot see your screen or access your files. If you say "fix my code" without pasting it, Claude can only respond generically. Always paste the exact relevant code block.

Bad
Fix my sorting function, it's returning the wrong order.
Good
Fix this Python function — it should sort ascending but returns descending: [paste code]
02 Using vague verbs — "write about", "do something with", "handle"
▾

Vague verbs force Claude to guess the format and depth. Use strong, specific verbs: summarize, compare, rewrite, extract, generate, classify.

Bad
Do something with this customer feedback list.
Good
Classify each piece of feedback as Bug / Feature Request / Compliment. Return a JSON array.
03 Loading one prompt with five different tasks
▾

Multi-goal prompts cause diluted, mediocre results across all tasks. Claude cannot give full attention to all five things simultaneously. Break them into separate messages in the same conversation.

Rule of thumb: one verb = one prompt. "Summarize AND translate AND suggest improvements AND format as JSON AND check for errors" should be 5 follow-up messages.

04 Assuming Claude knows your codebase, project, or company context
▾

Every new conversation starts with zero context. Claude doesn't know your tech stack, your team's conventions, your existing functions, or your business domain. You need to tell it.

Tip: create a short "context block" you paste at the start of each session — e.g. tech stack, coding conventions, relevant file structure. Takes 30 seconds, saves many revision cycles.

05 Not specifying format — then complaining the response isn't formatted right
▾

If you don't specify a format, Claude picks what it thinks is most appropriate — which may not be what you need. Always tell it: respond only in JSON, use a markdown table, numbered list, plain prose, etc.

Bad
List the pros and cons of React vs Vue.
Good
List pros and cons of React vs Vue as a markdown table with columns: Category | React | Vue.
06 Using "just" and "simply" — setting unrealistic expectations
▾

Words like "just", "simply", "quickly", and "easy" don't add information — they add pressure without precision. They can also cause Claude to skip important nuance or skip explaining trade-offs.

Replace them with specific scope or detail level: instead of "just summarize this," say "summarize in 3 bullet points, each under 15 words."

07 No constraints = no quality control
▾

Without constraints, Claude fills gaps with common defaults — which may not match your context. Constraints are not restrictions on Claude's ability; they're guardrails that eliminate the most common failure modes.

Negative constraints are especially powerful: "Do NOT use passive voice," "Do NOT include statistics unless cited," "Do NOT add an introduction paragraph." Each one removes a common mistake before it happens.

08 Giving up after one bad response instead of iterating
▾

A bad first response is diagnostic information, not failure. It tells you what context or constraint was missing. Use follow-up prompts to steer: The tone is too formal — rewrite it as casual, or The second section is too long, cut it to 2 sentences.

Claude holds full conversation context — you never have to repeat what you said earlier. Just describe what needs to change.
